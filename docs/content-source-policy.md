# Content Source Policy - BTP CloudEXCELON

This policy protects BTP from inaccurate claims while still allowing the sites to improve visually and strategically against competitor standards.

## Approved Content Sources

- BTP-owned websites, including the existing PraaS website and BTP Innovations public web properties.
- BTP Desktop folder assets, documents, PDFs, presentations, Word documents, service descriptions, logos, and marketing materials.
- Existing generated BTP website repositories and their documented audit findings.
- Conservative service copy derived from the approved BTP materials when the exact wording needs to be clearer for customers.

## Not Approved For Factual Copy

- Competitor websites.
- Generic marketing templates.
- AI-invented claims, numbers, awards, certifications, partnerships, case studies, or ownership status.
- Placeholder testimonials or performance metrics.

## Competitor Use Rule

Competitors may be reviewed for design quality, visual rhythm, service-page structure, CTA strategy, and information architecture. They must not be used as the factual source for what BTP claims to do, who BTP has served, what BTP has achieved, or what credentials BTP holds.

## Claim Standard

Every specific claim should be traceable to BTP-owned materials. If the source is unclear, keep the wording practical and non-specific, such as "structured delivery," "vendor coordination," "roadmap planning," or "support transition," rather than inventing a result.

## CloudEXCELON Content Boundaries

- Service lane: Cloud architecture and infrastructure built for scalable enterprise environments.
- Approved service capabilities: Cloud strategy, Cloud migration, Cloud optimization, Cloud infrastructure, Azure advisory, AWS advisory, Google Cloud advisory, Cloud operations.
- Approved outcome language: Scalable cloud foundations, Better cost control, Improved resilience, Cleaner operations.
- Approved deliverable language: Cloud readiness review, Target architecture, Migration and modernization roadmap, Cost optimization recommendations, Operations transition plan.

## Change Process

Before changing service copy, log the issue in `docs/website-audit.md`, identify the BTP source used, update the page, then run the QOS verification gate.
